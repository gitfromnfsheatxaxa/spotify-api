# Frontend-Design Skill

## Overview
Create distinctive, production-grade frontend interfaces with high design quality that avoids generic AI aesthetics.

## When to Use
Use this skill when the user asks to build:
- Web components or pages
- Full web applications
- UI interfaces and layouts
- Design systems and component libraries

## Design Thinking Process

### 1. Understand Context
- What problem does this interface solve?
- Who are the users?
- What are the technical constraints?

### 2. Choose Bold Aesthetic Direction
Pick an intentional direction:
- **Brutally Minimal**: Sparse, focused, maximum whitespace
- **Maximalist Chaos**: Dense, layered, overwhelming in a good way
- **Retro-Futuristic**: Y2K, vaporwave, synthwave aesthetics
- **Organic/Natural**: Biomorphic shapes, natural colors, soft edges
- **Luxury/Refined**: Premium feel, elegant typography, subtle details
- **Playful/Toy-like**: Fun, colorful, interactive, whimsical
- **Editorial/Magazine**: Typography-driven, grid-based, sophisticated
- **Brutalist/Raw**: Unpolished, honest, structural
- **Art Deco/Geometric**: Symmetry, patterns, ornamental details
- **Soft/Pastel**: Gentle colors, rounded shapes, comforting

### 3. Make Distinctive Choices

#### Typography
- Avoid: Inter, Arial, Roboto, system fonts
- Choose: Unexpected, characterful font pairings
- Pair: Display font + refined body font

#### Color & Theme
- Commit to cohesive aesthetic
- Use CSS variables for consistency
- Dominant colors with sharp accents
- Avoid: Evenly distributed timid palettes

#### Motion
- High-impact page load animations
- Staggered reveals with animation-delay
- Scroll-triggered interactions
- Surprise hover states
- CSS-first when possible

#### Spatial Composition
- Unexpected layouts
- Asymmetry and overlap
- Diagonal flow
- Grid-breaking elements
- Generous negative space OR controlled density

#### Backgrounds & Visual Details
- Gradient meshes
- Noise textures
- Geometric patterns
- Layered transparencies
- Dramatic shadows
- Custom cursors
- Grain overlays

## NEVER Use
- Generic AI-generated aesthetics
- Overused font families (Inter, Roboto, Arial)
- Purple gradient clichés on white
- Predictable layouts
- Cookie-cutter design patterns

## Implementation Standards
- Production-grade and functional code
- Visually striking and memorable
- Cohesive with clear aesthetic point-of-view
- Meticulously refined in every detail
- Match complexity to the vision (maximalist needs elaborate code, minimalist needs precision)

## Workflow
1. **Analyze** requirements and context
2. **Choose** bold aesthetic direction
3. **Design** with distinctive choices
4. **Implement** production-ready code
5. **Refine** every detail meticulously
6. **Verify** aesthetic cohesion

## Output Format
When generating code:
- Include all necessary CSS/styling
- Use distinctive fonts (import from Google Fonts or similar)
- Implement animations with Framer Motion or CSS
- Create visual interest through backgrounds, gradients, textures
- Add micro-interactions and hover states
- Ensure accessibility while maintaining aesthetic
